Plugin Components

boundary-segmenter

W2 — Jaccard-Cosine Boundary Segmentation. Weaver's defining engine. Clusters PostToolUse(Edit|Write) events into task boundaries via multi-modal distance (file-set Jaccard + token cosine + idle-time gap). Standalone: works without Hornet V1 by deriving token vectors from edit content. Upgraded automatically when Hornet V1 embeddings are available on the event bus.

Hooks
PostToolUse Edit|Write|MultiEdit → boundary-segment.sh (10s)
PreCompact * → checkpoint-clusters.sh (5s)
Skills
boundary-segmentation
Agents
boundary-detector (claude-opus-4-7, narrow)
Commands

branch-workflow

W3 — Workflow-Pattern Classifier. Infers the active branching model (GitHub Flow / Trunk-Based / GitFlow / Release Flow / Stacked Diffs / Unknown) from repo signals and suggests a branch name for the next task boundary. Per-subtree support for monorepos via .weaver/workflow-map.yaml.

Hooks

capability-memory

Provider capability registry — the 'memory' of how each git host behaves. Ships baseline data for 9 hosts (GitHub, GitLab, Bitbucket Cloud/DC, Azure DevOps, Gitea/Forgejo/Codeberg, AWS CodeCommit, SourceHut); refreshed nightly by CI and probed at runtime for GitLab self-managed version drift.

ci-reader

Read-only CI status + log streaming across GitHub Actions, GitLab CI, CircleCI, Jenkins, Buildkite, Drone/Woodpecker, Tekton, ArgoCD/FluxCD. Ships ci-registry.json — a declarative capability surface for all 10 systems (auth modes, rate limits, status/conclusion enums, webhook event taxonomy, known quirks) loaded at SessionStart. Normalizes per-system check payloads into a typed Check interface and emits weaver.ci.status.observed events. GitHub Actions fully implemented via gh CLI; 7 other systems stub and degrade to manual-handoff; ArgoCD/FluxCD are GitOps read-only drift detection.

W4 — Path-History Reviewer Routing + idempotent PR state machine.
